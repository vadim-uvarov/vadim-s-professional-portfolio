# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — Vite dev server on port 8080 (HMR overlay disabled).
- `npm run build` — production build to `dist/`.
- `npm run build:dev` — development-mode build (keeps the `lovable-tagger` plugin).
- `npm run lint` — ESLint over the repo (`eslint.config.js`, flat config, TS + React hooks).
- `npm test` — run Vitest once. `npm run test:watch` for watch mode.
- Single test: `npx vitest run src/path/to/file.test.ts` (or `-t "test name"`).
- Package manager: `npm` is the documented flow, but `bun.lockb` is also present. Pick one and don't mix.

## Coding Standards

### Commits
- Use commitizen format for commit messages
- Mention yourself as a co-author
- Commit along the way after completing each logical unit of work
- Combine related code changes into a single commit

### Function Naming
- Start all function and method names with action verbs.
- Prefer full variable names for readability (e.g., `response` instead of `resp`)
- Python: separate code from the inline comment with two spaces

### Formatting
- Python: use `ruff` and `mypy` for auto-formatting, linting and type checking
- Terraform: use `terraform fmt` for auto-formatting

## Pull Request

### Addressing PR comments
- When addressing PR comments, group similar changes and commit them into separate commits
- After finishing addressing PR comment, reply to the comment with a message that it's done 
    - provide summary on what changed
    - don't mention commit hash
    - mention that this reply was generated by Claude Code

## Documentation
- Keep readme.md up-to-date with the code
